# Agentic SDLC

> **Building Software with AI Teams**

This course teaches developers how to build and manage teams of AI agents that autonomously develop software. Move beyond single-agent coding assistance to orchestrated teams of specialists working in parallel.

## Before You Start

> **Critical:** Set up NATS JetStream BEFORE Hour 2. Students who skip this hit coordination failures in Week 1.

The course includes a "Before You Start" setup section. Complete it first:

| Component | Purpose |
|-----------|---------|
| **NATS JetStream** | Persistent message streams with complete audit trail |
| **Agent memory MCP** | Persistent context across sessions |

JetStream stores every message - you have a complete record of what agents said to each other, which is essential for debugging.

This will sound crude, but it's important:

When AI agents suddenly unblock your execution capability, most people experience what we call **"the diarrhea phase."** You've got years of backlogged projects and suddenly you can build them all. Right now. At once.

**This is incredibly exciting. It's also dangerous.**

> You still need to eat, sleep, and spend time with loved ones.

I've seen multiple developers hit manic episodes when they discover agent-assisted development. If you're prone to project-driven mania, **notify your loved ones NOW**. Tell them your teacher warned you this might trigger hyperfocus episodes.

### Set Hard Limits Before You Start

1. **Set a token budget** - Use Privacy.com to create merchant-locked cards with spending caps
2. **Set time boundaries** - Use timers, actually respect them
3. **Watch for warning signs** - Skipping meals, 3am coding sessions, "one more feature" loops

**The goal isn't to work 10x longer. The goal is to reclaim your time by working 10x faster in focused bursts, then stopping and doing something else with your life.**

## The Big Idea

**Traditional development:** One developer, one stream of work.

**Agentic development:** Clone your best engineers infinitely. Run 5 Roys at once. Let them coordinate via message queues instead of meetings. Watch them get better through memory and character development.
